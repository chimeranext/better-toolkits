# Lemon Squeezy: Integration Reference

## Overview

Lemon Squeezy is a **merchant of record** — they handle payments, taxes, invoicing, and compliance globally. You receive payouts minus their 5% + 50c fee. No need to handle VAT, GST, or sales tax yourself.

## When to Use in a Flutter App

Lemon Squeezy works best alongside (not replacing) store subscriptions:

| Scenario | Use Lemon Squeezy | Use RevenueCat |
|----------|-------------------|----------------|
| Digital content / licenses | Yes | No |
| SaaS web dashboard access | Yes | No |
| iOS/Android subscriptions | No | Yes |
| One-time digital products | Yes | Possible but overkill |
| API access tokens | Yes | No |
| Physical goods | No (use Shopify) | No |

### Apple/Google Store Guidelines

Important: If your app offers digital content consumed within the app, Apple and Google **require** you to use their in-app purchase systems (and pay their 15-30% commission). Lemon Squeezy should only be used for:
- Content consumed outside the app
- Web-based services
- B2B/enterprise purchases
- Physical goods and services
- Cross-platform licenses not tied to one device

## Architecture

```
Flutter App
  ├─ "Upgrade" button
  │   └─ Opens checkout URL in browser
  │       └─ Lemon Squeezy hosted checkout
  │           └─ Payment processed
  │               └─ Webhook → your backend
  │                   └─ Backend updates user record
  │                       └─ App checks entitlement via your API
  │
  └─ Deep link / universal link
      └─ Return to app after purchase
```

## API Overview

Base URL: `https://api.lemonsqueezy.com/v1`

Authentication: Bearer token (API key from Settings > API Keys)

### Key Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/checkouts` | POST | Create checkout session |
| `/orders` | GET | List/retrieve orders |
| `/subscriptions` | GET | List/retrieve subscriptions |
| `/subscriptions/{id}` | PATCH | Update subscription |
| `/license-keys/validate` | POST | Validate a license key |
| `/license-keys/activate` | POST | Activate license on device |
| `/license-keys/deactivate` | POST | Deactivate license |

### Create Checkout (from your backend)

```bash
curl -X POST "https://api.lemonsqueezy.com/v1/checkouts" \
  -H "Authorization: Bearer $LEMON_SQUEEZY_API_KEY" \
  -H "Content-Type: application/vnd.api+json" \
  -d '{
    "data": {
      "type": "checkouts",
      "attributes": {
        "checkout_data": {
          "custom": {
            "user_id": "user_123"
          }
        }
      },
      "relationships": {
        "store": {
          "data": { "type": "stores", "id": "YOUR_STORE_ID" }
        },
        "variant": {
          "data": { "type": "variants", "id": "YOUR_VARIANT_ID" }
        }
      }
    }
  }'
```

Response includes `url` — redirect user to this URL for payment.

### Validate License Key

```bash
curl -X POST "https://api.lemonsqueezy.com/v1/licenses/validate" \
  -H "Content-Type: application/json" \
  -d '{
    "license_key": "XXXXX-XXXXX-XXXXX-XXXXX",
    "instance_name": "user_device_id"
  }'
```

## Flutter Integration Pattern

### Opening Checkout

```dart
import 'package:url_launcher/url_launcher.dart';

class LemonSqueezyService {
  /// Open checkout in external browser
  /// The checkoutUrl should be generated by your backend
  static Future<void> openCheckout({
    required String checkoutUrl,
  }) async {
    final uri = Uri.parse(checkoutUrl);
    if (await canLaunchUrl(uri)) {
      await launchUrl(uri, mode: LaunchMode.externalApplication);
    }
  }
}
```

### License Validation (via your backend)

```dart
import 'dart:convert';
import 'package:http/http.dart' as http;

class LicenseService {
  static const _backendUrl = 'https://your-backend.com/api';

  /// Check if user has valid license
  static Future<bool> validateLicense(String userId) async {
    final response = await http.get(
      Uri.parse('$_backendUrl/license/validate?user_id=$userId'),
    );

    if (response.statusCode == 200) {
      final data = jsonDecode(response.body);
      return data['valid'] == true;
    }
    return false;
  }
}
```

## Webhook Events

Configure webhook URL in Lemon Squeezy > Settings > Webhooks:

**Key events:**
- `order_created` — new purchase
- `subscription_created` — new subscription
- `subscription_updated` — plan change, pause, resume
- `subscription_cancelled` — cancellation
- `subscription_expired` — subscription ended
- `subscription_payment_success` — renewal payment
- `subscription_payment_failed` — payment failed
- `license_key_created` — new license generated

### Webhook Payload Structure

```json
{
  "meta": {
    "event_name": "order_created",
    "custom_data": {
      "user_id": "user_123"
    }
  },
  "data": {
    "type": "orders",
    "id": "1234",
    "attributes": {
      "status": "paid",
      "total": 999,
      "currency": "USD",
      "user_email": "user@example.com"
    }
  }
}
```

### Webhook Signature Verification

Always verify webhook signatures:

```python
# Backend example (Python)
import hmac
import hashlib

def verify_webhook(payload: bytes, signature: str, secret: str) -> bool:
    expected = hmac.new(
        secret.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)
```

## Pricing Configuration

### Products in Lemon Squeezy

1. Go to Store > Products > + New Product
2. Set name, description, pricing:
   - **Subscription:** recurring (monthly/annual)
   - **Single payment:** one-time
   - **Pay what you want:** user chooses amount
   - **Free:** lead magnet / trial
3. Create variants for pricing tiers
4. Enable license keys if selling software licenses

### Tax Handling

Lemon Squeezy automatically:
- Calculates VAT/GST/sales tax by buyer location
- Adds tax to checkout (or includes it, configurable)
- Files tax returns in relevant jurisdictions
- Generates invoices for buyers

You don't need to register for VAT in the EU, UK, or other regions.
